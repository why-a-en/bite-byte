  <action>
    **Step 1: Install dnd-kit packages**

    ```bash
    pnpm add --filter @bite-byte/web @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
    ```

    **Step 2: Create menu server actions**

    Create `apps/web/src/actions/menu.ts`:
    - `createCategoryAction(venueId, prevState, formData)`: POST /venues/:venueId/categories with { name }
    - `updateCategoryAction(venueId, categoryId, prevState, formData)`: PATCH /venues/:venueId/categories/:id with { name }
    - `deleteCategoryAction(venueId, categoryId)`: DELETE /venues/:venueId/categories/:id
    - `reorderCategoriesAction(venueId, items: {id: string, sortOrder: number}[])`: PATCH /venues/:venueId/categories/reorder
    - (Item actions will be added in Task 2)

    All actions use the fetchApi helper from apps/web/src/lib/api.ts. All actions call `revalidatePath` after mutation to refresh server component data.

    **Step 3: Create CategoryList component (Client Component)**

    Create `apps/web/src/components/menu/category-list.tsx`:
    - Receives `initialCategories` as prop from server component
    - Wraps categories in `<DndContext>` + `<SortableContext>` with `verticalListSortingStrategy`
    - `onDragEnd`: use `arrayMove()` from @dnd-kit/sortable to reorder local state, then call `reorderCategoriesAction` with new sortOrder values (optimistic update)
    - Use stable UUIDs as item IDs (Prisma id field) — NOT array index (Pitfall 3 from RESEARCH.md)
    - "Add Category" button at bottom opens inline form or dialog
    - Each category rendered as SortableCategory component

    **Step 4: Create SortableCategory component (Client Component)**

    Create `apps/web/src/components/menu/sortable-category.tsx`:
    - Uses `useSortable` hook from @dnd-kit/sortable
    - Drag handle (grip icon from lucide-react)
    - Category name displayed (inline-editable or edit button → form)
    - Delete button (with confirmation)
    - Expand/collapse to show items within this category
    - When expanded, shows ItemCard components for each item + "Add Item" button

    Install lucide-react if not already installed by shadcn:
    ```bash
    pnpm add --filter @bite-byte/web lucide-react
    ```

    **Step 5: Create menu page (Server Component)**

    Create `apps/web/src/app/(dashboard)/venues/[venueId]/menu/page.tsx`:
    - Fetch categories with items: GET /venues/:venueId/categories (returns categories ordered by sortOrder with nested items)
    - Pass to CategoryList client component
    - Page title: "Menu Builder" or venue name + "Menu"
    - Empty state when no categories: "Add your first category to start building your menu"

  <action>
    **Step 1: Add item server actions to menu.ts**

    Add to `apps/web/src/actions/menu.ts`:
    - `createItemAction(venueId, categoryId, prevState, formData)`:
      - Extract name, description, price from formData
      - Extract photo File from formData (if provided)
      - If photo exists: upload via `put(`venues/${venueId}/items/${Date.now()}`, file, { access: 'public' })` from @vercel/blob. Get CDN URL.
      - POST /venues/:venueId/categories/:categoryId/items with { name, description, price, imageUrl }
      - revalidatePath

    - `updateItemAction(venueId, itemId, prevState, formData)`:
      - Same extraction pattern
      - If new photo: upload to Blob, get URL
      - PATCH /venues/:venueId/items/:id with updated fields

    - `deleteItemAction(venueId, itemId)`: DELETE /venues/:venueId/items/:id

    - `toggleAvailabilityAction(venueId, itemId, isAvailable: boolean)`:
      - PATCH /venues/:venueId/items/:id/availability with { isAvailable }
      - revalidatePath

    Photo upload note: use @vercel/blob `put()` with server upload pattern. If BLOB_READ_WRITE_TOKEN is not set (local dev without Vercel), gracefully skip upload and store null for imageUrl. Show a helpful message like "Image upload requires Vercel Blob configuration" instead of crashing.

    **Step 2: Create ItemForm component (Client Component)**

    Create `apps/web/src/components/menu/item-form.tsx`:
    - Used for both create and edit (receives optional existing item data)
    - Fields: name (required), description (optional textarea), price (required, number input with step="0.01"), photo (file input)
    - Photo upload UX:
      - File input styled as a dropzone or button
      - Client-side image preview (FileReader + img tag) before submit
      - Max file size check on client (warn if > 4 MB — Vercel Function 4.5 MB limit)
    - Use react-hook-form with Zod validation:
      - name: string, min 1 char
      - price: number, min 0.01
      - description: string, optional
    - Submit calls createItemAction or updateItemAction
    - Cancel button closes form
    - Display as dialog/modal (shadcn Dialog) or slide-out panel

    Install shadcn dialog if not done in Plan 04:
    ```bash
    cd apps/web && pnpm dlx shadcn@latest add dialog textarea
    ```

    **Step 3: Create ItemCard component (Client Component)**

    Create `apps/web/src/components/menu/item-card.tsx`:
    - Displays item: thumbnail photo (or placeholder), name, description (truncated), price
    - Availability toggle: shadcn Switch component — calls toggleAvailabilityAction on change
    - Visual indicator when unavailable: grayed out, "Unavailable" badge
    - Edit button → opens ItemForm in edit mode (pre-filled)
    - Delete button with confirmation (shadcn AlertDialog)
    - Card layout: horizontal on desktop (image left, details right), stacked on mobile

    Install shadcn switch:
    ```bash
    cd apps/web && pnpm dlx shadcn@latest add switch
    ```

    **Step 4: Wire ItemCard into SortableCategory**

    Update `sortable-category.tsx` to render ItemCard components for each item in the category. Show "Add Item" button that opens ItemForm in create mode.

    Design priorities:
    - Items within a category should be visually grouped
    - The "86 an item" toggle should be highly visible and fast (one click)
    - Photo thumbnails should be small (~64x64 or ~80x80) to not overwhelm the list
    - Empty category state: "No items yet. Add your first item."
