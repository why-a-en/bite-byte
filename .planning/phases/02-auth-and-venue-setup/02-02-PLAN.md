  <action>
    Create VenuesModule with controller and service.

    **VenuesController** (all endpoints use `@UseGuards(JwtAuthGuard)`):
    - `POST /venues` — create venue. Accepts { name, slug, logoUrl?, paymentMode? }. Sets ownerId from req.user.userId. Validates slug uniqueness (Prisma unique constraint handles this; catch P2002 and return 409 Conflict).
    - `GET /venues` — list all venues for the authenticated owner. Returns array with categories count.
    - `GET /venues/:id` — get single venue by ID, only if owned by authenticated user. 404 if not found or not owned.
    - `PATCH /venues/:id` — update venue settings. Accepts partial { name?, slug?, logoUrl?, paymentMode? }. Only if owned by authenticated user. Validate slug uniqueness on update too.
    - `DELETE /venues/:id` — delete venue and cascade. Only if owned.

    **VenuesService**:
    - Inject PrismaService.
    - ALL queries include `WHERE ownerId = userId` — this is application-layer security, NOT RLS (see RESEARCH.md Pattern 7). The CLS `VENUE_ID` / RLS is for Phase 3 customer routes.
    - create(userId, dto): `prisma.venue.create({ data: { ...dto, ownerId: userId } })`
    - findAllForOwner(userId): `prisma.venue.findMany({ where: { ownerId: userId }, orderBy: { createdAt: 'asc' } })`
    - findOneForOwner(id, userId): `prisma.venue.findFirst({ where: { id, ownerId: userId } })` — returns null if not owned
    - update(id, userId, dto): findFirst to verify ownership, then update
    - remove(id, userId): findFirst to verify ownership, then delete

    Extract userId from request: the JwtStrategy.validate() returns { userId: payload.sub, email }, which Passport attaches as `req.user`. In the controller, use a custom decorator or `@Req() req` to access `req.user.userId`.

    Add VenuesModule to AppModule imports.
  </action>

  </verify>
  <done>Venue CRUD endpoints exist, all protected by JwtAuthGuard, all scoped to authenticated owner via ownerId WHERE clause</done>

  <action>
    **CategoriesModule** (all endpoints `@UseGuards(JwtAuthGuard)`):

    CategoriesController:
    - `POST /venues/:venueId/categories` — create category. Accepts { name, sortOrder? }. Verify venue is owned by authenticated user first (call venuesService.findOneForOwner or inline Prisma check). 403/404 if not owned.
    - `GET /venues/:venueId/categories` — list categories for venue, ordered by sortOrder. Include items in each category.
    - `PATCH /venues/:venueId/categories/:id` — update category (name, sortOrder).
    - `DELETE /venues/:venueId/categories/:id` — delete category (cascades items? or reject if items exist — prefer reject with 409 to prevent accidental data loss).
    - `PATCH /venues/:venueId/categories/reorder` — accepts array `[{ id: string, sortOrder: number }]`, bulk updates sortOrder in a transaction. This is the DnD persistence endpoint.

    CategoriesService:
    - All queries verify venue ownership: `prisma.venue.findFirst({ where: { id: venueId, ownerId: userId } })` before any category operation.
    - reorder(venueId, userId, items: {id, sortOrder}[]): verify venue ownership, then `prisma.$transaction(items.map(i => prisma.menuCategory.update({ where: { id: i.id }, data: { sortOrder: i.sortOrder } })))`.

    **ItemsModule** (all endpoints `@UseGuards(JwtAuthGuard)`):

    ItemsController:
    - `POST /venues/:venueId/categories/:categoryId/items` — create item. Accepts { name, description?, price, imageUrl?, sortOrder? }. Verify venue ownership. Set venueId on the item (MenuItem has venueId for RLS in Phase 3).
    - `GET /venues/:venueId/items` — list all items for venue (across all categories), ordered by category sortOrder then item sortOrder.
    - `PATCH /venues/:venueId/items/:id` — update item (name, description, price, imageUrl, sortOrder, categoryId).
    - `DELETE /venues/:venueId/items/:id` — delete item. Verify venue ownership.
    - `PATCH /venues/:venueId/items/:id/availability` — toggle isAvailable. Accepts { isAvailable: boolean }. This is the "86 an item" endpoint (MENU-06).

    ItemsService:
    - Same ownership verification pattern as CategoriesService.
    - Price is stored as Decimal in Prisma — accept price as number from client, Prisma handles Decimal conversion.

    Import both CategoriesModule and ItemsModule into CategoriesModule and ItemsModule respectively need VenuesModule for ownership checks. Options: (a) Import VenuesModule and use VenuesService, or (b) inline the ownership check in each service. Prefer (b) — inline `prisma.venue.findFirst({ where: { id: venueId, ownerId: userId } })` in each service to avoid circular deps.

    Add CategoriesModule and ItemsModule to AppModule imports.
