  <action>
    1. Create `apps/api/src/analytics/analytics.service.ts`:
       - Inject `PrismaService`.

       - `getRevenue(venueId: string)`:
         - Calculate startOfDay, startOfWeek (Sunday), startOfMonth using `new Date()`.
         - Run three `prisma.order.aggregate()` calls in `Promise.all`:
           - WHERE: `{ venueId, status: { not: 'CANCELLED' }, createdAt: { gte: startOf* } }`
           - Select: `_sum: { totalAmount: true }`
         - Return `{ today: number, week: number, month: number }` — convert null _sum to 0.

       - `getTopItems(venueId: string, take = 5)`:
         - Use `prisma.orderItem.groupBy()`:
           - `by: ['menuItemId', 'itemNameAtOrder']`
           - `where: { order: { venueId, status: { not: 'CANCELLED' } } }`
           - `_sum: { quantity: true }`
           - `orderBy: { _sum: { quantity: 'desc' } }`
           - `take`
         - Return array of `{ name: string, count: number }` — map `r.itemNameAtOrder` and `r._sum.quantity ?? 0`.

       - `getDailyVolume(venueId: string, days = 7)`:
         - Calculate `since = new Date(); since.setDate(since.getDate() - days)`.
         - Use `prisma.$queryRaw` with tagged template:
           ```sql
           SELECT DATE(created_at) as date, COUNT(*) as count
           FROM orders
           WHERE venue_id = ${venueId}::uuid
             AND status != 'CANCELLED'
             AND created_at >= ${since}
           GROUP BY DATE(created_at)
           ORDER BY date ASC
           ```
         - CRITICAL: Convert BigInt to Number — `rows.map(r => ({ date: String(r.date), count: Number(r.count) }))`. PostgreSQL COUNT returns BigInt in Prisma $queryRaw. JSON.stringify will throw "Do not know how to serialize a BigInt" if not converted.
         - Return array of `{ date: string, count: number }`.

       - `getHistory(venueId: string, from?: string, to?: string, page = 1, pageSize = 20)`:
         - Build WHERE clause: `{ venueId, status: { in: ['COMPLETED', 'CANCELLED'] as const } }`.
         - If `from` or `to` provided, add `createdAt: { gte: new Date(from), lte: new Date(to) }` accordingly.
         - Run `Promise.all` with `findMany` (include items, orderBy createdAt desc, skip/take for pagination) and `count` for total.
         - Return `{ orders, total, page, pageSize }`.

       - `verifyVenueOwnership(venueId: string, userId: string)`:
         - Query `prisma.venue.findUnique({ where: { id: venueId }, select: { ownerId: true } })`.
         - If not found, throw NotFoundException.
         - If `ownerId !== userId`, throw ForbiddenException.
         - This is called by the controller before each analytics method.

    2. Create `apps/api/src/analytics/analytics.controller.ts`:
       - Decorate with `@Controller('venues/:venueId')` and `@UseGuards(JwtAuthGuard)`.
       - Inject `AnalyticsService`.
       - Use `@Req() req` to get `req.user.userId` for ownership verification.

       - `@Get('analytics/revenue')` — call verifyVenueOwnership, then getRevenue.
       - `@Get('analytics/top-items')` — call verifyVenueOwnership, then getTopItems. Accept optional `@Query('take')` param (default 5).
       - `@Get('analytics/daily-volume')` — call verifyVenueOwnership, then getDailyVolume. Accept optional `@Query('days')` param (default 7).
       - `@Get('orders/history')` — call verifyVenueOwnership, then getHistory. Accept `@Query('from')`, `@Query('to')`, `@Query('page')`, `@Query('pageSize')` params.

       NOTE: The route `venues/:venueId/orders/history` must NOT collide with the public OrdersController at `public/venues/:slug/orders/:orderId`. These are on completely different path prefixes (`venues/:venueId` vs `public/venues/:slug`) so there is no collision.

    3. Create `apps/api/src/analytics/analytics.module.ts`:
       - Standard NestJS module: controllers: [AnalyticsController], providers: [AnalyticsService].

    4. Update `apps/api/src/app.module.ts`:
       - Import `AnalyticsModule` and add to the imports array.

    AVOID: Forgetting to convert BigInt to Number in getDailyVolume — this is the most common Prisma $queryRaw pitfall (Pitfall 4 from RESEARCH.md). Do NOT use Prisma groupBy for daily aggregation — it cannot extract date parts from timestamps (Pitfall 8).
