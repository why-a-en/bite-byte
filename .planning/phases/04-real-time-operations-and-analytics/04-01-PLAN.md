       - Decorate with `@WebSocketGateway({ cors: { origin: process.env.CORS_ORIGIN ?? 'http://localhost:3000', credentials: true } })` — NO port argument (shares HTTP port for Railway single-port deployment).
       - Implement `OnGatewayInit` interface.
       - Inject `JwtService` (already global from AuthModule).
       - In `afterInit(server: Server)`: add `server.use()` middleware for TWO-TIER AUTH:
         - If `socket.handshake?.auth?.token` exists, verify with `jwtService.verify(token)`, set `socket.data.userId = payload.sub`, `socket.data.isAuthenticated = true`. If verification fails, call `next(new Error('Unauthorized'))`.
         - If NO token is provided, allow through: set `socket.data.isAuthenticated = false`, call `next()`. This permits anonymous customer connections for order status rooms.
       - `@SubscribeMessage('join:venue')` handler: MUST verify `socket.data.isAuthenticated === true` AND query Prisma to check `venue.ownerId === socket.data.userId` before calling `client.join('venue:' + venueId)`. If check fails, return `{ error: 'Forbidden' }` — do NOT join the room.
       - `@SubscribeMessage('join:order')` handler: anyone can join (no auth check), call `client.join('order:' + orderId)`.
       - Public methods for service to call:
         - `emitNewOrder(venueId: string, order: object)` — emits `'order:new'` to `venue:{venueId}` room
         - `emitOrderUpdated(venueId: string, orderId: string, update: object)` — emits `'order:updated'` to BOTH `venue:{venueId}` AND `order:{orderId}` rooms
       - Inject `PrismaService` for the venue ownership check in `handleJoinVenue`.

    3. Update `apps/api/src/orders/orders.module.ts`:

       - Also export `OrdersGateway` (WebhooksModule will need it for Stripe order emissions).

    AVOID: Specifying a port number in @WebSocketGateway — Railway exposes one port and the WS must share the HTTP port. Do NOT use Guards for WS auth — Guards don't block initial connections in Socket.IO; use server.use() middleware in afterInit() instead.
