  <how-to-verify>
**Prerequisites:**
1. `pnpm dev` from monorepo root (starts Next.js on :3000, NestJS on :3001)
2. Have a venue with menu items created in the dashboard (from Phase 2)
3. For Stripe testing: Set real `STRIPE_SECRET_KEY`, `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`, and optionally run `stripe listen --forward-to localhost:3001/api/webhooks/stripe` for webhook testing

**Test flow A — Pay at Counter venue (minimum viable test, no Stripe required):**

1. Visit `http://localhost:3000/menu/[your-venue-slug]`
   - Expected: Menu loads with categories and items in stacked rows — NO login required
   - Expected: Item photos visible as thumbnails on the right

2. Tap an item row
   - Expected: Bottom sheet slides up from bottom with full photo, description, price, "Add to Cart" button

3. Tap "Add to Cart"
   - Expected: Sheet closes; floating black pill button appears at bottom center showing "1 · View Cart · £X.XX"
   - Expected: Unavailable items are dimmed and non-tappable

4. Add 2-3 more items; tap "View Cart"
   - Expected: Slide-up drawer with cart contents; +/- buttons per item; total at bottom

5. In cart drawer, change a quantity and remove an item
   - Expected: Quantities update in real-time; removing last item hides floating button

6. Add items back; refresh the page (F5 or mobile refresh)
   - Expected: Cart still shows the items (localStorage persistence — ORDR-04)

7. In cart drawer, tap "Checkout"
   - Expected: Navigates to /menu/[slug]/checkout

8. On checkout page: enter your name; if venue is PAC: tap "Place Order"
   - Expected: Order created; navigates to /menu/[slug]/order/[orderId]

9. On order status page:
   - Expected: Reference code displayed (e.g. "A7K2X9B4")
   - Expected: Status steps show "Order Received" as active
   - Expected: Open Network tab — every ~5 seconds a request to /api/public/venues/[slug]/orders/[id]
   - Expected: No polling requests after status reaches READY or COMPLETED (change status manually via DB or Phase 4 to test this)

10. Close browser tab; reopen and navigate back to same URL
    - Expected: Order status page still loads correctly (ORDR-10 — URL-based recovery)

**Test flow B — Stripe prepay (requires Stripe test keys + stripe listen):**

1. Set venue paymentMode to PREPAY_REQUIRED
2. Follow steps 1-7 above
3. On checkout page: enter name; tap "Proceed to Payment"
   - Expected: Stripe Payment Element renders (card field, Apple Pay if available)
4. Enter Stripe test card: `4242 4242 4242 4242`, any future expiry, any CVC
5. Tap "Pay Now"
   - Expected: Payment confirms without redirect; navigates to order status page
   - Expected: Order status shows "Pending" initially (PENDING_PAYMENT → webhook fires → RECEIVED)
   - If stripe listen is running: within a few seconds status updates to "Order Received"

**BOTH payment mode check:**
1. Set venue paymentMode to BOTH
2. On checkout page: verify two radio buttons appear — "Pay now" and "Pay at counter"

**Mobile check:**
- Open Chrome DevTools → Toggle Device Emulation → iPhone 14
- Repeat test flow A
- Expected: Bottom sheet and cart drawer feel native with swipe-to-close gesture
